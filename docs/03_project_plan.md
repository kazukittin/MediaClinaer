# 開発計画書

## 1. 開発方式

ウォーターフォール開発で進める。各工程の終わりに成果物を確認し、次工程へ進む。

## 2. 確定した方針

- 対象OSはWindowsのみ
- 初期版はPython + PySide6のデスクトップアプリ
- Portable配布前提
- 初期版は完全削除なし、隔離フォルダへの移動のみ
- 類似動画検出は初期版から外す
- NAS共有フォルダをスキャン対象に含める
- 隔離フォルダはPCローカル上の1か所にまとめる
- NAS上のファイルもPCローカルへ隔離する
- 検出結果と解析結果はキャッシュとして保存する
- キャッシュは無期限に保持する
- 隔離フォルダとキャッシュフォルダはアプリフォルダ配下に作成する
- 元ファイルと隔離ファイルは完全パスで紐づける
- アプリフォルダ配下へ書き込めない場合は処理を開始しない
- 対応画像形式にJPGを含める
- 長いパスは可能な限り扱い、処理できないファイルはスキップしてログに記録する
- 検出結果、解析キャッシュ、隔離履歴はSQLiteに保存する
- 設定ファイルはJSONに保存する
- SQLiteスキーマは詳細設計書の内容で確定
- 設定ファイル項目は詳細設計書の内容で確定
- ログは1行JSON形式で確定
- 隔離処理はコピー検証後に元ファイルを削除する二段階方式で確定
- 実装モジュール構成は詳細設計書の内容で確定
- UIと処理本体は分離する
- 初期実装は土台、DB、スキャン、完全重複、最小UI、隔離、類似画像、ブレ画像の順で進める
- NASアクセス失敗時の例外処理は詳細設計書の内容で確定
- テストデータは自動テスト用と手動テスト用に分ける
- 単体テストと手動テストの範囲は詳細設計書の内容で確定

## 3. 工程

| 工程 | 内容 | 成果物 |
| --- | --- | --- |

- 対象OSをWindowsに確定する
